# Network Shared Profile Readonly Detail Design

Date: 2026-04-16

## Goal

让“网络共享库”中的共享配置也支持“查看详情”，但详情页保持只读，不能编辑、不能保存。

## User-Facing Outcome

- 网络共享库卡片新增“查看详情”入口。
- 点击后进入现有详情页样式的详情界面。
- 共享配置的名称、备注、`auth.json`、`config.toml` 都可查看。
- 该详情界面所有输入区域只读，保存类按钮隐藏或禁用，并明确提示“网络共享配置仅可查看，不能编辑”。
- “安装并应用”继续保留在网络共享库卡片中，不与详情页编辑能力混淆。

## Non-Goals

- 不新增远程配置的编辑、回传或覆盖能力。
- 不改动 Tauri 后端 profile 存储模型。
- 不改变本地 profile 的现有编辑流程。

## Options Considered

### Option A: 为网络共享库单独做一个新详情页

优点：
- 语义清晰。

缺点：
- 重复 UI 和状态结构。
- 后续本地/远程详情字段容易漂移。

### Option B: 复用现有详情页并增加只读模式

优点：
- 改动范围最小。
- 复用现有布局和字段渲染。
- 本地与网络共享详情表现保持一致。

缺点：
- 需要给编辑器状态增加来源和只读控制。

### Recommended: Option B

采用统一详情页，给编辑器状态增加“网络共享来源”和“只读”标记。网络共享详情从远程接口加载文本内容后写入统一详情状态，界面根据只读标记禁用输入并隐藏保存动作。

## Implementation Notes

- 前端类型增加网络共享详情文档结构。
- 新增打开网络共享详情的方法，按需请求远程 profile 详情和文件内容。
- `renderEditorPage()` 根据只读标记切换标题、副标题、提示文案与按钮显隐。
- 为网络共享页签增加回归测试，覆盖：
  - 卡片存在“查看详情”按钮
  - 点击后进入详情页
  - 文本区域带 `disabled`
  - 页面不展示保存动作
